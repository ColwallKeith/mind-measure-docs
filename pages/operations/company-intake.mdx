# Company Onboarding Intake

A single questionnaire to capture everything we need from a new institutional
customer before we open the Mind Measure CMS. Send this to the primary
contact and ask them to return it complete — every section maps directly to
one of the CMS tabs, so a fully-completed intake means setup is a
twenty-minute job rather than a two-week back-and-forth.

The document is structured so it can be sent as a Word doc, a PDF, a Google
Form, a Typeform, or pasted into a Notion page without losing meaning. Each
section header tells the receiver what it's for; each field is marked
**Required** or *Optional* and shows the format we expect.

> Internal note: the right-hand column for each field shows where it lands in
> the Mind Measure platform (CMS tab → DB column). Hide that column when
> sharing the intake externally.

## 4. Organisational structure

Mind Measure groups employees by **Division** (business unit / line of
service) and then by **Department** (team / practice). Cohort analytics
become much richer once this is populated. If the company's nomenclature
differs, see [Section 6 — Employee profile fields](#6-employee-profile-fields)
to relabel.

### 4.1 Divisions

For each division, give us:

- Name (e.g. "Consulting")
- Short name (e.g. "Cons") — used in narrow dashboard cards
- Approx headcount
- Departments / practices underneath (see 4.2)

### 4.2 Departments

For each department within a division:

- Name (e.g. "Strategy &amp; Operations")
- Approx headcount

A blank departments list is fine — we can collect that information later
once employees self-assign in the mobile app.

### 4.3 Offices / sites

List every physical office or work site the pilot covers:

| Field | Required? | Example |
|---|---|---|
| Office name | **Required** | "Emaar Square, Dubai" |
| City | **Required** | "Dubai" |
| Country | **Required** | "UAE" |
| Approx headcount | *Optional* | 1,400 |
| Address | *Optional* | Used for local-resource mapping |

them for a few profile fields. The defaults (below) cover most companies,

| Field | Default label | Default options | Change requested? |
|---|---|---|---|
| Division | "Business unit" | from your 4.1 list | |
| Department | "Department" | from your 4.2 list | |
| Seniority | "Level" | Junior · Mid-level · Senior · Lead/Manager · Director+ | |
| Work mode | "Work mode" | Office · Hybrid · Remote | |
| Office | "Primary office" | from your 4.3 list | |

If your company uses a different ladder (e.g. PwC uses Associate → Senior

manager, secondment status — let us know and we'll wire them in.

## 7. Wellbeing & welfare services

This section is the most important. It populates the in-app **Help**
screen, the Jodie agent's knowledge base, and the welfare escalation flow
described in the [Wellbeing Protection Policy](/operations/wellbeing-protection-policy-pro)
and [Hierarchy of Help](/operations/hierarchy-of-help).

### 7.1 Internal support

| Service | Required? | What we need |
|---|---|---|
| **Employee Assistance Programme (EAP)** | **Required** | Provider name, 24/7 phone, web portal URL, login instructions |
| **HR support contact** | **Required** | Name, email, phone, hours |
| **Occupational health** | *Optional* | Provider, contact, how to access |
| **Mental Health First Aiders** | *Optional* | Named list with contact (Slack, email, phone) |
| **Manager support / line manager escalation** | *Optional* | One-paragraph description of how an employee escalates a concern |
| **Internal wellbeing programmes** | *Optional* | Yoga, mindfulness, peer-support groups — name + how to join |

### 7.2 External crisis resources

We include sensible defaults per country (UK: Samaritans 116 123, etc.).
If your company prefers different services or operates in regions our
defaults don't cover (e.g. UAE, KSA), list them here:

| Field | Example |
|---|---|
| Service name | "Estijaba — UAE Mental Wellness Line" |
| Phone | 8001717 |
| Hours | 24/7 |
| Languages | Arabic, English |
| Web | https://… |
| Notes | Free for residents of Abu Dhabi |

### 7.3 Hierarchy of help

When Mind Measure detects a Level-3+ score, the in-app interstitial offers
the employee a graded list of help options. Tell us your preferred order
for this company. Default Pro order is:

1. Your line manager / HR business partner
2. Your EAP (24/7)
3. Mental Health First Aider
4. National crisis line (country-specific)
5. Emergency services (999 / 911 / 112 / local)

Override any step or insert company-specific options here.

## 8. Jodie — custom check-in questions *(optional)*

Mind Measure's voice agent Jodie can ask a small number of
company-specific questions during the daily check-in. Use these to surface
issues unique to the company or sector. Examples:

- Workload sustainability around quarter-end
- Sentiment toward an active change programme
- Specific high-risk projects or client engagements

| Field | Example |
|---|---|
| Question | "How are you finding the workload as we approach the close of Q3?" |
| When to ask | Always · Weekly · Only in the first month |
| Tone | Neutral · Encouraging · Direct |
| Surfaces on dashboard? | Yes — as its own cohort filter |

Limit: **five custom questions** per company on the standard tier.

## Internal: import workflow

Once the completed intake is returned:

1. **Create the company shell** via Superuser → Companies → New. Paste the
   data from Sections 1–3.
2. **Org structure** (Sections 4.1–4.3) — paste into the Organisational
   Structure tab in the CMS. Divisions JSONB, Departments JSONB, Offices
   JSONB.
3. **Profile fields** (Section 6) — Employee Profile Fields tab. If
   customised, override the default config.
4. **Welfare services** (Sections 7.1, 7.2, 7.3) — Emergency Resources tab,
   Hierarchy of Help tab.
5. **Allowed domains** (Section 5) — Settings → `allowed_email_domains`.
   Also add to `DOMAIN_OVERRIDES` in `CompanyResolver.ts` if you need an
   immediate non-DB override.
6. **Jodie custom questions** (Section 8) — Institutional Agents tab. The
   Jodie agent is regenerated automatically.
7. **Brand assets** (Section 3) — upload via the Settings tab. Logo files
   land in S3 (`mindmeasure-user-content-459338929203`).
8. **Smoke test** — sign in to the mobile app with a test email matching
   the allowed domain, complete the baseline, verify cohort analytics
   surface the correct division/department lists.
9. **Status flip** — once the smoke test passes, move company status from
   `planning` → `launched` (this enables real-data display in the dashboard
   instead of the placeholder zero-state).
