## User Flow

### Authentication

```
App Launch → Authenticated? ──No──→ Sign In (AWS Cognito)
                │
               Yes
                │
        Email Verified? ──No──→ Email Verification
                │
               Yes
                │
        Profile Complete? ──No──→ Profile Creation + University Assignment
                │
               Yes
                │
        Baseline Complete? ──No──→ Baseline Assessment
                │
               Yes
                │
           Dashboard
```

### First-Time User Journey

1. **Sign up** with email/password (Cognito)
2. **Verify email** via confirmation code
3. **Create profile** — first name, display name, auto-assigned to university
4. **Baseline assessment** — Jodie conducts initial structured conversation (6 questions)
5. **Post-baseline dashboard** — shows baseline score as benchmark

### Returning User Journey

1. **Sign in** → recognises returning user
2. **Dashboard** — shows latest wellbeing data, trends, insights
3. **Check-in** — conversational check-in with Jodie (open-ended, ~6–8 exchanges)
4. **Results** — updated dashboard with new scores, themes, summary

## Jodie Voice Agent (ElevenLabs)

The **Jodie** agent is the conversational AI that conducts check-ins. Each product line has its own ElevenLabs agent with a dedicated prompt.

### SDK Integration

```typescript
import { useConversation } from '@elevenlabs/react';

const conversation = useConversation({
  onConnect: () => { /* agent connected */ },
  onDisconnect: () => { /* session ended */ },
  onMessage: (message) => { /* process message */ },
});

await conversation.startSession({
  agentId: import.meta.env.VITE_ELEVENLABS_CHECKIN_AGENT_ID,
  dynamicVariables: {
    user_name: profile.firstName,
    last_summary: lastSession?.summary || '',
    last_themes: lastSession?.themes?.join(', ') || '',
    checkin_count: String(profile.checkinCount || 0),
  },
  overrides: {
    agent: {
      firstMessage: `Hey ${profile.firstName}, good to chat again...`,
    },
    tts: {
      voiceId: '...',
    },
  },
});
```

### Agent Configuration

| Setting | Value | Notes |
|---|---|---|
| First message override | ON | Must be enabled in ElevenLabs dashboard |
| Voice stability override | ON | Must be enabled |
| Voice similarity override | ON | Must be enabled |
| Default personality | OFF | Conflicts with prompt |
| Scribe Realtime ASR | v2.1 | Speech recognition model |

### Prompt Files

Prompts are managed locally (not in the codebase):

- `jodie_prompt_v2_uni.md` — University agent prompt
- `jodie_prompt_v2_pro.md` — Pro agent prompt

Key prompt rules:
- Numbers written as words ("seven" not "7") to avoid punctuation errors in speech
- Pacing: 6–8 exchanges before asking for mood score
- Must explore main topic with follow-ups before changing subject
- Mood score asked on 1–10 scale using words

## Multi-Modal Analysis Pipeline

When a check-in ends, three analyses run in parallel:

### 1. Text Analysis (AWS Bedrock)

```
Transcript → /api/bedrock/analyze-text → Claude 3.5 Sonnet
```

Outputs:
- `conversation_summary` — unique summary of what was discussed
- `sentiment_score` — 0–100 overall sentiment
- `driver_positive` — array of positive themes
- `driver_negative` — array of concerns
- `risk_flags` — safeguarding indicators
- `mood_score` — extracted mood score (1–10)

### 2. Visual Analysis (AWS Rekognition)

```
Video frames (captured every 2s) → /api/rekognition/analyze-frame → emotion detection
```

Outputs:
- Dominant emotion per frame
- Face detection rate
- Engagement level

### 3. Audio Analysis (Client-Side)

```
Web Audio API → speech rate, voice quality, emotional tone
```

Extracted client-side during conversation, no server round-trip.

### Fusion

```
V2 Fusion: 70% text + 15% audio + 15% visual → final wellbeing score
```

Results stored in `fusion_outputs` table and displayed on the dashboard.

## Dashboard

The dashboard shows:

- **Wellbeing Score** — current score with trend indicator
- **Latest Insight** — Bedrock-generated conversation summary
- **Previous Insight** — summary from the prior check-in
- **Enjoying / Worry Cards** — positive drivers and concerns
- **Mood Trend** — 7/30/90-day mood history chart
- **Buddy System** — accountability partner status
- **Content** — articles and nudges from the content hub
- **Help** — emergency resources and support contacts

### Check-In Button

Prominent "Start Check-In" button launches the Jodie agent. Camera and microphone permissions requested on first use.

## Capacitor (Native iOS)

The app is built as a web app and wrapped with Capacitor for native iOS distribution.

### Build Process

```bash
npm run build           # Vite production build
npx cap sync            # Sync web assets to iOS project
npx cap open ios        # Open in Xcode
```

Then archive and upload to TestFlight from Xcode.

### Native Features Used

- Camera access (for visual analysis)
- Microphone access (for voice conversation)
- Haptic feedback
- Status bar styling

### CORS Considerations

Mobile apps running via Capacitor use origin `capacitor://localhost`. All API CORS configurations must include this origin alongside the web domains.

## API Endpoints

All mobile API endpoints are Vercel serverless functions with inlined database and auth code (due to Vercel/Vite bundling constraints — see `.cursor/rules/vercel-api-inlining.mdc`).

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/database/select` | POST | Read from database |
| `/api/database/insert` | POST | Write to database |
| `/api/database/update` | POST | Update records |
| `/api/database/health-check` | GET | Database connectivity check |
| `/api/bedrock/analyze-text` | POST | Transcript analysis |
| `/api/rekognition/analyze-frame` | POST | Visual emotion analysis |
| `/api/buddies` | GET | List buddies |
| `/api/buddies/invite` | POST | Send buddy invitation |

### Inlining Rule

Vercel's Vite builder does not bundle shared files from `_lib/` directories in serverless functions. All database pool setup, Cognito JWT verification, and retry logic must be **inlined** directly into each API file. Do not import from shared `_lib/` files.
