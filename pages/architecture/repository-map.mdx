## University Repositories

### mind-measure-core

The university admin platform. Houses the superuser panel, institutional dashboards, and all Vercel serverless API functions.

| | |
|---|---|
| **GitHub** | [Mind-Measure/mind-measure-core](https://github.com/Mind-Measure/mind-measure-core) |
| **Domain** | [admin.mindmeasure.co.uk](https://admin.mindmeasure.co.uk) |
| **Stack** | React + Vite + TypeScript |
| **Deploy** | Vercel (auto-deploy from `main`) |

**Contains:**
- Superuser control panel (overview, universities, users, marketing, security, costs, system, resources, docs)
- University-scoped institutional dashboards
- Vercel serverless API functions (`/api/*`)
- Ecosystem health audit (`/api/system/ecosystem-audit`)
- Authentication (AWS Cognito + superuser 6-digit code flow)
- University onboarding and configuration

### mind-measure-mobile-final

The university student mobile app, deployed as web (Vercel) and native iOS (Capacitor).

| | |
|---|---|
| **GitHub** | mind-measure-mobile-final |
| **Domain** | [mobile.mindmeasure.app](https://mobile.mindmeasure.app) (web) + iOS via TestFlight |
| **Stack** | React + Vite + Capacitor + TypeScript |
| **Deploy** | Vercel (web) + Xcode/TestFlight (iOS) |

**Contains:**
- Student dashboard with wellbeing insights
- Jodie voice agent check-in (ElevenLabs conversational AI)
- Baseline assessment flow
- Buddy system
- Content page, help page, profile

### mind-measure-university-hub

Content management for university administrators. Accessed via the institutional dashboard — no standalone login.

| | |
|---|---|
| **GitHub** | [Mind-Measure/mind-measure-university-hub](https://github.com/Mind-Measure/mind-measure-university-hub) |
| **Domain** | [hub.mindmeasure.co.uk](https://hub.mindmeasure.co.uk) |
| **Stack** | React + Vite + TypeScript |
| **Deploy** | Vercel (auto-deploy from `main`) |

**Contains:**
- Article editor (Tiptap)
- Nudge manager
- Content pool browser
- Student insights (aggregated)

## Deployment Pipeline

All repositories follow the same pattern:

1. Push to `main` on GitHub
2. Vercel auto-detects the push and builds
3. If build succeeds, deployed to production domain
4. Custom domain serves the new version

**Never use `npx vercel --prod` or the Vercel CLI.** See [Deployment Guide](/quick-start/deployment) for the full protocol.
