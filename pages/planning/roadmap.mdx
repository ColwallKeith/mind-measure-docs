**Last Updated**: 28 February 2026

---

## Active Planning

### Institutional Agents

**Priority**: High  
**Status**: Spec complete — [view full spec](/planning/institutional-agents)

Each university/company gets its own ElevenLabs Jodie agent with a personalised knowledge base compiled from CMS onboarding data. Custom questions become a premium revenue feature.

**Key milestones**:
- Phase 1: Agent ID lookup table and mobile app integration
- Phase 2: Knowledge base auto-compilation from CMS
- Phase 3: Custom questions with natural conversation weaving
- Phase 4: Aggregated reporting dashboard for institutions

**Revenue impact**: Custom questions create a unique data feedback loop — anonymised, natural-language wellbeing data on topics institutions care about, gathered through conversation rather than surveys.

---

### Streak Rewards

**Priority**: High  
**Status**: Concept — spec to be written

Users are rewarded for regular check-ins with digital vouchers to spend with partner retailers or on campus. Incentivises consistent engagement and creates a tangible value exchange for users who maintain check-in habits.

**Key milestones**:
- Phase 1: Streak tracking infrastructure and reward tiers
- Phase 2: Partner retailer integration and voucher generation
- Phase 3: On-campus redemption system
- Phase 4: Analytics dashboard for reward programme performance

**Revenue impact**: Partner retailer relationships create sponsorship revenue. Consistent engagement improves data quality and institutional value proposition.

---

### Student Ambassador Platform

**Priority**: High  
**Status**: Concept — spec to be written

A social media assets library and student ambassador signup, logging, and management system. Ambassadors receive bonuses based on users signing up and regularly using the platform.

**Key milestones**:
- Phase 1: Ambassador signup and onboarding flow
- Phase 2: Social media assets library (branded content, templates, guidelines)
- Phase 3: Referral tracking — link sign-ups to ambassadors and track ongoing engagement
- Phase 4: Ambassador dashboard — login, view sign-ups, monthly payouts, performance metrics
- Phase 5: Admin management — approve ambassadors, set bonus tiers, process payouts

**Revenue impact**: Organic user acquisition through peer networks. Ambassador programme reduces marketing spend while increasing authentic engagement.

## Feature Ideas

### Investor Dashboard

**Priority**: Medium  
**Origin**: Investor page exists on mindmeasure.co.uk — no dashboard or designs yet

Build a dedicated investor-facing dashboard (separate from superuser portal or within it):
- User growth and engagement trends
- University/company adoption metrics
- Revenue and cost overview
- Platform health summary
- Requires design work — no Figma designs exist for this

---

### Downloadable Wellbeing Reports (Enhanced)

**Priority**: Medium  
**Status**: V1 deployed — detailed reports covering 14/30/90-day periods with executive summary, transcript snippets, and charts  
**Blocker**: Further development requires professional input from therapists to ensure clinical appropriateness

Planned enhancements:
- Therapist-reviewed content guidance and language
- Actionable wellbeing suggestions (curated by clinicians, not AI-generated)
- Comparison to personal baseline with clinical context
- Institution-branded report templates
- Exportable data for clinical handoffs

---

### Real-Time Risk Escalation

**Priority**: High  
**Origin**: Bedrock analysis already scores risk

Build on the existing risk detection in Bedrock analysis:
- Real-time alerts to designated welfare contacts when high-risk indicators are detected
- Configurable thresholds per institution
- Escalation workflow (alert → acknowledge → follow-up → resolve)
- Audit trail for safeguarding compliance
- Integration with institutional safeguarding systems

---

### Text-Based Check-In Alternative

**Priority**: Medium  
**Origin**: Not all users want voice check-ins

Offer a text-based chat alternative to voice:
- Same Jodie persona and conversation flow
- Uses ElevenLabs text-based agent or direct LLM integration
- Lower data usage, works in quiet environments
- User preference stored in settings (once Settings Phase 2 is built)

---

### Multi-Language Support

**Priority**: Low (future)  
**Origin**: International student populations

- Jodie agent speaks in the user's preferred language
- ElevenLabs supports multiple languages for voice
- Bedrock analysis handles non-English transcripts
- UI localisation for mobile app
- Knowledge base documents in multiple languages

---

### Peer Support Network

**Priority**: Low (future)  
**Origin**: Extension of buddy system

Evolve the buddy system into a broader peer support network:
- Trained peer supporters matched to students
- Guided conversation prompts for supporters
- Feedback loop to institution on peer support usage
- Integration with institutional peer mentoring schemes

---

### API for Institutional Systems

**Priority**: Low (future)  
**Origin**: Enterprise integration requests

Expose anonymised, aggregated data via API:
- Wellbeing trend data for institutional dashboards
- Integration with student records systems
- SSO integration for institutional login
- Webhook notifications for risk escalation

## How to Use This Document

- **Add ideas freely** — capture everything, filter later
- **Move to "Active Planning"** when a spec is being written
- **Link to full specs** in the Planning section when they exist
- **Park ideas** that aren't ready but shouldn't be forgotten
- **When a feature is complete**: remove it from Active Planning / Feature Ideas, add it to the Completed Features table with a link to a write-up in `/planning/completed/`, and ensure it is listed in [Future Development](/future-development)
- **Never delete** — move to Parked with a note instead
- Keep the "Last Updated" date current
