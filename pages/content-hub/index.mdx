# University Content Hub

## Overview

The University Content Hub is a standalone Next.js application that provides university administrators with a dedicated space to create, manage, and share wellbeing content for their students. It is accessed exclusively via the Institutional Dashboard (admin.mindmeasure.co.uk) вҖ” there is no standalone login.

| | |
|---|---|
| **Repository** | [mind-measure-university-hub](https://github.com/Mind-Measure/mind-measure-university-hub) |
| **Production URL** | [hub.mindmeasure.co.uk](https://hub.mindmeasure.co.uk) |
| **Framework** | Next.js 14 (App Router) |
| **Deployment** | Vercel (auto-deploy from GitHub) |
| **Database** | AWS Aurora PostgreSQL (shared with core) |

## Why a Separate Application?

The Content Hub was deliberately separated from `mind-measure-core` to prevent codebase contamination. The core repository handles the Institutional Dashboard, superuser controls, and API layer. Content management has distinct dependencies (rich text editor, media upload, content pool) that would bloat the core if co-located.

Separation also enables independent deployment cycles вҖ” content features ship without risking the admin dashboard or assessment pipeline.

## Features

### Content Management

University admins can create and manage long-form wellbeing articles for their students. The editor uses [Tiptap](https://tiptap.dev/) вҖ” the same rich text editor used in the Mind Measure Marketing CMS вҖ” providing a consistent authoring experience.

Articles support:
- Title, subtitle, and author attribution
- Cover images (uploaded to S3 via the Marketing CMS proxy)
- Categories and tags (stored as JSONB)
- Draft / Published / Archived status
- "Share to Content Pool" toggle for cross-university visibility
- **Cohort targeting** вҖ” direct articles to specific student groups
- **Pin to top** вҖ” pin articles to the top of the student feed until a specified date

### Content Pool

Three content streams are available to university admins:

| Stream | Description |
|---|---|
| **Your Content** | Articles created by this university |
| **Public Pool** | Articles shared by other universities and seeded by the Mind Measure team |
| **Popular Content** | Trending articles from the community, ranked by engagement |

Content from the Mind Measure Marketing CMS can be pushed to the public pool via the `targetSites` field on blog posts.

### Nudge Manager

Nudges are short, prescriptive notifications displayed to students on the mobile app. They link to university wellbeing pages and are prioritised by urgency.

**Nudge fields:**
- Headline
- One-line description
- Category (determines colour variant and priority)
- Date/time (for events)
- Link URL
- Status (active / archived)

**Categories and priority:**
| Category | Colour | Priority | Example |
|---|---|---|---|
| Emergency | Bittersweet (red) | Highest | Crisis support changes |
| Deadline | Buttercup (amber) | High | Exam registration deadline |
| Workshop | Sinbad (teal) | Medium | Mindfulness session |
| Social | Lilac (purple) | Lower | Coffee morning in JCR |
| General | Pampas (cream) | Base | General wellbeing tip |

Nudges are stored in the `universities.nudges` JSONB column and sorted automatically by category priority. A maximum of 5 active nudges are displayed as a swipeable carousel on the mobile app.

### Cohort Targeting

Both articles and nudges can be targeted to specific student cohorts. The Audience Selector is placed at the top of content editing forms to encourage administrators to consider their audience before creating content.

**How it works:**

1. Toggle between "Whole University" (default) and "Specific Cohort"
2. When targeting is active, a row of cascading multi-select dropdowns appears
3. Select one or more schools вҖ” the Department dropdown then shows only departments within those schools
4. Add further filters: year group, accommodation type, hall of residence, study mode
5. A recap bar displays the selected criteria alongside a live count of matching students

**Targeting dimensions:**

| Dimension | Profile Field | Notes |
|-----------|--------------|-------|
| School / Faculty | `school` | Top-level academic grouping |
| Department | `course` | Cascades from school selection |
| Year | `year_of_study` | Year 1 through Postgraduate |
| Accommodation | `living_situation` | On Campus, Off Campus, At Home |
| Hall of Residence | `hall_of_residence` | Only shown when "On Campus" is selected |
| Study Mode | `study_mode` | Full-time, Part-time |

Targeting uses AND logic вҖ” a student must match all specified criteria to see the content. Untargeted content is always visible to all students.

**Data storage:**
- Nudges: `targeting` field within the JSONB nudge object in `universities.nudges`
- Articles: `targeting` JSONB column on the `university_articles` table

**Mobile filtering:**
The mobile API proxies (`api/nudges/active.ts`, `api/content/pool.ts`) perform server-side filtering. Each content item's targeting criteria are checked against the requesting user's profile. Only matching content is returned.

### Pin to Top

Any nudge or article can be pinned to the top of the student feed until a specified date. This ensures time-sensitive content (exam support weeks, event promotions, deadline reminders) is seen first.

- Toggle "Pin to top" on the content editing form
- Set an expiry date (defaults to 7 days from today)
- Pinned content sorts above all non-pinned content in both the nudge carousel and article feed
- When the date passes, the content drops back to its normal position вҖ” no manual action required

**Data storage:**
- Nudges: `pinnedUntil` field within the JSONB nudge object
- Articles: `pinned_until` TIMESTAMPTZ column on the `university_articles` table

### Student Insights

Aggregated, anonymised data from student check-ins provides a "Top 10" of student worries and interests, drawn from `fusion_outputs` analysis. This data is available both locally (university-specific) and nationally, providing inspiration for content creation.

## Technical Architecture

### Database

The Hub shares the Aurora PostgreSQL instance with the core platform:

```sql
-- Articles table
CREATE TABLE university_articles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  university_id TEXT NOT NULL,
  title TEXT NOT NULL,
  subtitle TEXT,
  slug TEXT NOT NULL,
  content TEXT,
  cover_image TEXT,
  author TEXT,
  status TEXT DEFAULT 'draft',
  categories JSONB DEFAULT '[]',
  tags JSONB DEFAULT '[]',
  shared_to_pool BOOLEAN DEFAULT FALSE,
  targeting JSONB DEFAULT NULL,
  pinned_until TIMESTAMPTZ DEFAULT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Nudges stored on the universities table as JSONB
-- Each nudge object includes: id, title, description, category,
-- eventDate, pinnedUntil, linkUrl, targeting, status, createdAt, updatedAt
ALTER TABLE universities
  ADD COLUMN IF NOT EXISTS nudges JSONB DEFAULT '[]';
```

### API Routes

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/auth/session` | GET | Token-based session creation |
| `/api/auth/logout` | GET | Session teardown, redirect to CMS |
| `/api/content` | GET, POST | List and create articles |
| `/api/content/[id]` | GET, PUT | Read and update a specific article |
| `/api/content/pool` | GET | Fetch shared pool articles from Marketing CMS |
| `/api/nudges` | GET, POST, PUT, DELETE | CRUD for university nudges |
| `/api/university-structure` | GET | Schools, departments, halls for the audience selector |
| `/api/audience-count` | GET | Live count of profiles matching targeting criteria |
| `/api/insights` | GET | Aggregated student worries and interests |
| `/api/upload` | POST | Proxy image uploads to S3 via Marketing CMS |
