# Marketing Sites Overview

## Purpose

Mind Measure marketing websites follow a design-led, content-unified approach:

- **Figma** – Single source of truth for page layouts (home, about, contact, science, how it works)
- **Marketing CMS** – One place for blog content, feeding both marketing sites and mobile app articles
- **No Sanity** – Page structure comes from Figma; blogs come from Marketing CMS
- **Easy expansion** – Add Mind Measure Pro (workplace) and new pages without duplicating content systems

## Content Sources

### Page Layouts (Static)

- **Source:** Figma designs
- **Flow:** Design in Figma → Figma Make (React output) or hand-code → Deploy
- **Pages:** Home, About, How it works, Contact, The science, etc.

### Blog & Articles (Dynamic)

- **Source:** [Marketing CMS](https://marketing.mindmeasure.co.uk) (GitHub: [mind-measure-marketing-cms](https://github.com/Mind-Measure/mind-measure-marketing-cms))
- **Flow:** Create post in CMS → Publish → Marketing sites fetch via public API at build → Mobile app receives via University deployments
- **One write, multiple outputs:** Same blog content appears on marketing sites and as in-app articles

## Tech Stack

### Core Technologies

- **React** – Figma Make outputs React; marketing sites use React-based stack
- **TypeScript** – Type safety
- **Tailwind CSS** – Styling
- **Marketing CMS** – Blog and article content (Prisma + PostgreSQL)

### Design

- **Figma** – Page layouts, design system, Mind Measure Pro designs
- **Figma Make** – Design-to-code (React output)

### Deployment

- **Vercel** – Marketing sites and Marketing CMS
- **GitHub** – Version control

## Key Workflows

### Content Editors

1. Log in to [marketing.mindmeasure.co.uk](https://marketing.mindmeasure.co.uk)
2. Create/edit blog posts
3. Set target sites (university, student, pro)
4. Publish – content flows to marketing sites and (optionally) mobile app

### Designers

1. Design pages in Figma (home, about, contact, etc.)
2. Use shared design system across sites
3. Export via Figma Make or hand off to developers
4. New pages follow same pattern

### Developers

1. Build static pages from Figma designs
2. Wire blog routes to Marketing CMS public API
3. Deploy to Vercel
4. Marketing sites rebuild when content is published (webhook or manual)

## Migration from Sanity

- **Deprecated:** Sanity is no longer the content source for marketing sites
- **Replaced by:** Figma (pages) + Marketing CMS (blogs)
- **Action:** Add public API endpoints to Marketing CMS; point marketing sites to them; remove Sanity
