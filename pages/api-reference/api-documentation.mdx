# API Documentation

# Mind Measure API Reference

## Overview

The Mind Measure API provides secure, scalable endpoints for mental health assessment processing, data management, and university administration. The API is built on AWS serverless architecture with comprehensive authentication and rate limiting.

## Base URLs

### Assessment Engine API (Multimodal Assessment)
```
Production: https://ej3hig3gwl.execute-api.eu-west-2.amazonaws.com/v1/
Region: eu-west-2 (London)
Deployed: November 28, 2025
```

### Mobile App API (Vercel Serverless)
```
Production: https://mobile.mindmeasure.app/api
Endpoints: /api/database/*, /api/storage/*, /api/auth/*
```

### Admin Dashboard API (Vercel Serverless)
```
Production: https://admin.mindmeasure.co.uk/api
Endpoints: Admin-specific endpoints
```

## Authentication

### JWT Bearer Tokens
All API endpoints require authentication via JWT tokens issued by AWS Cognito.

```http
Authorization: Bearer <jwt_token>
```

### Token Structure
```typescript
interface JWTPayload {
  sub: string;           // User ID
  email: string;         // User email
  email_verified: boolean;
  aud: string;           // Client ID
  iss: string;           // Cognito issuer
  exp: number;           // Expiration timestamp
  iat: number;           // Issued at timestamp
}
```

### Getting Tokens
```typescript
// Using AWS Amplify
import { fetchAuthSession } from 'aws-amplify/auth';

const session = await fetchAuthSession();
const token = session.tokens?.accessToken?.toString();

// API call with token
const response = await fetch('/api/endpoint', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
```

## Rate Limiting

### Assessment Engine API
```yaml
/checkins/start: 10 requests/minute per user
/checkins/:id/complete: 10 requests/minute per user
/checkins/:id: 60 requests/minute per user (polling)
```

### Vercel Database APIs
```yaml
/api/database/select: 100 requests/minute per user
/api/database/insert: 50 requests/minute per user
/api/database/update: 50 requests/minute per user
/api/database/delete: 20 requests/minute per user
File Upload: 10 requests/minute per user
```

## Error Handling

### Standard Error Response
```typescript
interface ErrorResponse {
  error: string;           // Error message
  code?: string;          // Error code

}
```

### HTTP Status Codes
```yaml
200: Success
400: Bad Request - Invalid parameters
401: Unauthorized - Invalid or missing token
403: Forbidden - Insufficient permissions
404: Not Found - Resource not found
429: Too Many Requests - Rate limit exceeded
500: Internal Server Error - Server error
```

## Assessment Engine API

The Assessment Engine provides multimodal wellbeing assessment using 23 audio features, 18 visual features, and 16 text features. Deployed November 28, 2025.

### POST /v1/checkins/start

Creates a new check-in and generates presigned S3 upload URLs for media files.

**Request**
```typescript
POST https://ej3hig3gwl.execute-api.eu-west-2.amazonaws.com/v1/checkins/start
Headers: {
  Authorization: Bearer <cognito_jwt_token>,
  Content-Type: application/json
}
Body: {
  type: "daily" | "baseline" | "adhoc"
}
```

**Response**
```typescript
{
  checkInId: string;  // UUID
  type: "daily" | "baseline" | "adhoc";
  uploadUrls: {
    audio: {
      url: string;      // Presigned S3 PUT URL
      bucket: string;   // S3 bucket name
      key: string;      // S3 object key
    };
    images: Array<{
      url: string;
      bucket: string;
      key: string;
    }>;  // 8-12 image URLs
  };
}
```

### POST /v1/checkins/:id/complete

Marks media uploads complete and starts the processing pipeline.

**Request**
```typescript
POST https://ej3hig3gwl.execute-api.eu-west-2.amazonaws.com/v1/checkins/:checkInId/complete
Headers: {
  Authorization: Bearer <cognito_jwt_token>,
  Content-Type: application/json
}
Body: {
  type?: "daily" | "baseline" | "adhoc"  // Optional, defaults to existing type
}
```

**Response**
```typescript
{
  checkInId: string;
  status: "PROCESSING";
}
```

### GET /v1/checkins/:id

Retrieves check-in status and results (when complete).

**Request**
```typescript
GET https://ej3hig3gwl.execute-api.eu-west-2.amazonaws.com/v1/checkins/:checkInId
Headers: {
  Authorization: Bearer <cognito_jwt_token>
}
```

**Response** (Status: PROCESSING)
```typescript
{
  checkInId: string;
  status: "PROCESSING";
}
```

**Response** (Status: COMPLETE)
```typescript
{
  checkInId: string;
  status: "COMPLETE";
  score: {
    mindMeasureScore: number;        // 0-100
    uncertainty: number;             // 0-1
    directionOfChange: "better" | "same" | "worse";
    riskLevel: "none" | "mild" | "moderate" | "high";
  };
  modalities: {
    audio?: {
      score: number;      // -1 to 1
      confidence: number; // 0-1
    };
    visual?: {
      score: number;
      confidence: number;
    };
    text?: {
      score: number;
      confidence: number;
    };
  };
}
```

**Response** (Status: FAILED)
```typescript
{
  checkInId: string;
  status: "FAILED" | "QUALITY_FAILED";
  error?: string;  // Human-readable error message
}
```

### Example Usage

```typescript
// 1. Start check-in
const startResponse = await fetch(
  'https://ej3hig3gwl.execute-api.eu-west-2.amazonaws.com/v1/checkins/start',
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${jwtToken}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ type: 'daily' })
  }
);
const { checkInId, uploadUrls } = await startResponse.json();

// 2. Upload media to S3
await fetch(uploadUrls.audio.url, { method: 'PUT', body: audioBlob });
await Promise.all(
  uploadUrls.images.map(img => 
    fetch(img.url, { method: 'PUT', body: imageBlob })
  )
);

// 3. Complete check-in
await fetch(
  `https://ej3hig3gwl.execute-api.eu-west-2.amazonaws.com/v1/checkins/${checkInId}/complete`,
  {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${jwtToken}` }
  }
);

// 4. Poll for results
const result = await pollForResult(checkInId, jwtToken);
console.log('Mind Measure Score:', result.score.mindMeasureScore);
```

### Audio Analysis

#### POST /analyze-audio

Processes voice patterns and emotional tone analysis from conversation data.

**Request**
```typescript
interface AudioAnalysisRequest {
  sessionId: string;
  audioData: {
    conversation_duration: number;    // Duration in seconds
    speech_rate: number;             // Words per minute ratio
    voice_quality: string;           // 'clear' | 'strained' | 'muffled'
    emotional_tone: string;          // 'positive' | 'negative' | 'neutral'
    mood_score_1_10: number;        // Self-reported mood (1-10)
    transcript_length: number;       // Character count
  };
}
```

**Response**
```typescript
interface AudioAnalysisResponse {
  sessionId: string;
  analysis: {
    vocal_stress_indicators: number;     // 0-1 scale
    speech_pattern_analysis: {
      pace_variation: number;
      pause_frequency: number;
      volume_consistency: number;
    };
    emotional_markers: string[];         // Detected emotional indicators
    confidence_score: number;            // Analysis confidence (0-1)
  };
  processing_time: number;               // Processing time in ms
}
```


### Fusion Score Calculation

#### POST /calculate-mind-measure

Calculates the final Mind Measure score using multi-modal fusion algorithm.

**Request**
```typescript
interface FusionRequest {
  sessionId: string;                   // Session with completed analyses
}
```

**Response**
```typescript
interface FusionResponse {
  sessionId: string;
  fusion_score: {
    final_score: number;               // Final score (0-100)
    confidence: number;                // Overall confidence (0-1)
    component_scores: {
      audio_score: number;             // Audio component (0-100)
      visual_score: number;            // Visual component (0-100)
      text_score: number;              // Text component (0-100)
      passive_score?: number;          // Passive data if available
    };
    risk_assessment: {
      level: 'low' | 'medium' | 'high';
      indicators: string[];            // Risk factors identified
      recommendations: string[];       // Suggested actions
    };
    trend_analysis?: {
      compared_to_baseline: number;    // Difference from baseline
      trend_direction: 'improving' | 'stable' | 'declining';
      significance: 'low' | 'medium' | 'high';
    };
  };
  metadata: {
    algorithm_version: string;
    processing_time: number;
    data_quality_score: number;
  };
}
```

**Example**
```bash

  https://4xg1jsjh7k.execute-api.eu-west-2.amazonaws.com/dev/calculate-mind-measure \
  -H "Authorization: Bearer <jwt_token>" \
  -H "Content-Type: application/json" \
```

```python
import requests
import json
from typing import Dict, Any, Optional

class MindMeasureAPI:
    def __init__(self, base_url: str, token: str):
        self.base_url = base_url
        self.token = token
        self.headers = {
            'Authorization': f'Bearer {token}',
            'Content-Type': 'application/json'
        }
    
    def _request(self, endpoint: str, method: str = 'GET', data: Optional[Dict] = None):
        url = f"{self.base_url}{endpoint}"
        
        if method == 'POST':
            response = requests.post(url, headers=self.headers, json=data)
        else:
            response = requests.get(url, headers=self.headers, params=data)
        
        response.raise_for_status()
        return response.json()
    
    def select(self, table: str, **options):
        return self._request('/api/database/select', 'POST', {
            'table': table,

        return self._request('/analyze-audio', 'POST', {
            'sessionId': session_id,
            'audioData': audio_data
        })
```
